Data levels

How the data nests

The whole database is built around one idea: bigger things contain smaller things. A mouse has eyes; an eye’s retina gives you cells; a cell’s recording is sliced into trials. DataJoint encodes that nesting with primary keys — a child row carries its parent’s key, so “this epoch’s cell’s animal’s genotype” is always answerable.

The main ladder

This is the path most physiology questions travel. Each arrow names the field that ties the two levels together.

🐭Animal sln_animal.Animal
One mouse. Everything traces back here. Keyed by animal_id.
linked by animal_id
👀Eye sln_animal.Eye
The left or right eye of that mouse. Its retina is what you record from.
the retina becomes a recording source
◉Cell sln_symphony.ExperimentCell
One neuron you patched or recorded, e.g. 101323Ac3. Keyed by cell_number.
its trials are grouped into a dataset
▤Dataset sln_symphony.Dataset
A curated bundle of epochs for one analysis (e.g. all the “spots multi size” trials for this cell).
a dataset is made of many epochs
⋮Epoch sln_symphony.ExperimentEpoch
A single trial — one presentation of the stimulus. Keyed by epoch_id.
spikes detected in it become a spike train
☰Spike train sln_symphony.SpikeTrain
The detected spikes for that epoch — the thing most spike analyses count.
Why this matters for queries Because every level carries the keys above it, you can restrict at any level and the rest follows. Restrict datasets to one animal_id, or restrict spike trains to one cell_number — DataJoint walks the links for you. You never write a JOIN by hand.

Animal sln_animal.Animal

The root of everything. One row per mouse, with an auto-assigned animal_id, date of birth, sex, and its strain. The colony apps write here; you mostly read from it to group results by genotype or age.

Eye sln_animal.Eye

Each animal has up to a few eyes (Left, Right, or unknown). On the recording side, an eye’s retina shows up as a source in a Symphony file (sln_symphony.ExperimentRetina), which links back here by animal_id. This is the bridge between the colony world and the recording world.

Cell sln_symphony.ExperimentCell · sln_cell.Cell

One recorded neuron. In a file it’s an ExperimentCell with a cell_number and its position (x, y microns from the optic nerve). The lab-wide sln_cell.Cell gives that same neuron a permanent cell_unid and is where a cell type gets attached. Cell names read like 101323Ac3 — date, rig, cell number.

Dataset sln_symphony.Dataset

A curated group of epochs a person picked out for one analysis — for example, every good “spots multi size” trial from this cell. Datasets are the unit almost every result table is built on (one row of results per dataset). Made in the DataGrouper app; the pairing of epochs to datasets lives in sln_symphony.DatasetEpoch.

Epoch sln_symphony.ExperimentEpoch

A single trial: one presentation of the stimulus, with a start time and duration. The raw trace for an epoch lives in ExperimentEpochChannel; the stimulus parameters for that trial live in the protocol’s …ep table.

Spike train sln_symphony.SpikeTrain

After spike detection (manual or automatic), each epoch channel gets a SpikeTrain with a spike_count and the spike_indices. Spike-based analyses like SMS_CA and MovingBar_CA depend on this existing.

The other view: the raw file

The ladder above is the biological story. The raw .h5 file organizes the same recordings a little differently — worth recognizing, because block- and epoch-level tables follow this shape, and it’s where protocol_name comes from.

📁Experiment sln_symphony.Experiment
One .h5 file = one recording day on one rig, e.g. 101323A. Keyed by file_name.
a file contains one or more sources
◎Source sln_symphony.ExperimentSource
A thing being recorded: a retina, a cell, or a cell pair. Keyed by file_name + source_id.
a source runs several epoch groups
◫Epoch group sln_symphony.ExperimentEpochGroup
A labelled stretch of the session, e.g. “control” vs “drug”.
each group holds epoch blocks
▤Epoch block sln_symphony.ExperimentEpochBlock
One run of one protocol — this is where protocol_name lives.
each block is many epochs
⋮Epoch sln_symphony.ExperimentEpoch
A single trial. The same epoch a dataset groups for analysis.
The join point A dataset and a raw epoch block both ultimately group epochs. That’s the seam where the curated analysis world meets the raw recording world — and why analyses so often join DatasetEpoch with an epoch or block table.
