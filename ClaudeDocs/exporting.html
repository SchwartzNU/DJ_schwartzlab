Getting data out

Exporting data

Once you’ve found or computed something, there are two ways to take it with you: point-and-click from DataGrouper, or a couple of lines at the MATLAB command line. It helps to think in terms of two questions — what you’re exporting, and what form you want it in.

What you can export

  • A data group — the datasets you selected, with their metadata, and optionally the raw traces and spike times.
  • A computed result — the rows of one analysis’s result table (e.g. sln_results.DatasetSMSCA).
  • A saved query’s output — its datasets or its analysis results, straight from QueryBrowser.

What form it can take

  • The MATLAB workspace — a struct (and table) you can poke at interactively. Best for “let me just look at this now.”
  • A standard MATLAB table — a table object you can sort, join, and writetable to CSV/Excel.
  • A .mat file — for picking the same data group back up later in MATLAB, or handing to another MATLAB user.
  • An .h5 (HDF5) file — language-agnostic and good for large arrays: read it from Python (h5py), share it, or archive it.

1 · Export a data group from DataGrouper

Select the datasets

Query and select the datasets you want in the Browser.

Choose what to bundle

Tick Include raw traces and/or Include spike times if you need the signals, not just the metadata. (Leaving them off keeps the export small.)

Pick the destination

The File / Command line rocker decides where it goes: File saves a .mat, and Command line drops data_group and data_group_struct into your MATLAB workspace.

Export

Click Export Full DataGroup.

2 · Fetch a result table at the command line

Any analysis result is a table you can restrict and pull directly (see Writing an analysis for how they’re built):

in MATLAB
% Grab one analysis's results, restricted to the cells you care about:
R = sln_results.DatasetSMSCA & 'source_id = 3';

% Pull every column into a MATLAB struct array (one element per row):
s = fetch(R, '*');

% ...or just the columns you need, as arrays:
[rate, psth] = fetchn(R, 'baseline_rate_hz', 'sms_psth');

fetch(R, '*') gives you a struct array (one element per row); fetchn gives you each column as its own array.

3 · Turn it into a standard MATLAB table

A struct array is fine, but a table is easier to slice, join, and save:

in MATLAB
% Turn the fetched struct array into a standard MATLAB table:
T = struct2table(s);

% Now it behaves like any table -- sort, group, join, or write out:
T = sortrows(T, 'baseline_rate_hz');
writetable(T, 'sms_results.csv');     % -> CSV / Excel, no DataJoint needed
Why bother Once it’s a table you get writetable for CSV/Excel, easy filtering, and clean joins with other tables — and columns that held vectors or images (longblob) come through as cell columns you can index into.

4 · Write it to an HDF5 (.h5) file

For sharing outside MATLAB or archiving, HDF5 is the format to reach for. From DataGrouper:

  • To HDF5 — writes the single stored result you’ve selected in Results in database to an .h5 file. (To command line sends that same result to the workspace instead.)
  • Export Results / Export Selection — hand the current selection to the H5_Exporter, which builds a structured .h5 of the whole data group.
Reading it back An .h5 opens anywhere — h5read in MATLAB, or h5py in Python — which makes it the right choice when a collaborator isn’t on the DataJoint pipeline.

5 · From a saved query (QueryBrowser)

If the data lives behind a saved query, you don’t need to rebuild it. In QueryBrowser, select the query and click Fetch to get its datasets in the workspace, or pick a result table and click Fetch result — the Result to switch sends it to the command line or a file.

Which should I use? Quick look now → command line. Tabular analysis or a spreadsheet → a MATLAB table / CSV. Reuse in MATLAB later → .mat. Sharing with non-MATLAB tools or archiving → .h5.
