DataGrouper V2

One window for almost everything: searching the database, grouping epochs into datasets, tagging, cell typing, running analyses, plotting, and exporting. Launch it by typing DataGrouper_V2 in the MATLAB Command Window and pressing Enter. Wait until the attribute list on the left fills in and the window centers itself before clicking anything.

The interface, in seven regions

  1. 1
    Attributes & controls. The left strip: the View level selector, the attribute list you build queries from, and the buttons for running analyses, exporting, cell typing, Curator mode, and choosing a project.
  2. 2
    Query construction. Where your query is assembled, condition by condition, with the Logical operators helper and a live Results count in its lower corners.
  3. 3
    New user parameter. The form for creating a brand-new tag — name, type, options, and description. Creating one builds a new UserParam… table.
  4. 4
    Browser. The list of items at the current view level (experiments, cells, datasets…). Selecting rows here is what analyses, exports, and cell-typing act on.
  5. 5
    Results in database. Stored analysis results for whatever you’ve selected — the jumping-off point for plotting and To HDF5 export.
  6. 6
    Plots. The plotter: pick a plot type, filter by stimulus condition, and save figures as PNG / PDF / FIG.
  7. 7
    Details. A summary of the selected item, plus the To Curator and To Spike Detector buttons at Cell level.
The one idea to hold onto Everything works off the current view level and what you have selected in the Browser. Set the view level (Experiment → Animal → Eye → Cell → Dataset → Epoch), run a query to fill the Browser, select some rows, then act on them — analyze, tag, type, or export.

1 · Build a query

A query is how you find your data. You assemble it in the Query construction panel from attributes (the column names in the list on the left — see the Tables page for what they mean).

Pick the view level

Set View level to what you want back — usually Dataset or Cell.

Choose an attribute and add it

Select an attribute (e.g. cell_type), then click Add condition with attribute. Fill in the value. Leave out quotation marks. For wildcards choose IS LIKE and use % (any characters) or _ (one character).

Combine conditions

Extra attributes are joined with AND automatically. For an OR, use the Logical operators buttons: a true condition AND ANY OF ( A , B ).

Run

Click Run. The Results box shows how many experiments / animals / cells / datasets matched, and the Browser fills in. Clear query starts over; Delete last element removes the most recent piece.

Curator mode Flip the Curator mode switch to also see experiments and cells that have no datasets yet — useful for finding un-curated or un-typed data. You can’t query saved results or run dataset analyses in this mode, and cell types show incorrectly, so turn it off before assigning types.

2 · Tags (user parameters)

Tags let you attach your own labels to data — “drug present”, “good example cell”, a hand-scored quality, and so on. Under the hood each tag is a small table named UserParam<Level><Name> (you’ll spot these on the Tables page), attached to whatever level you made it at.

Add an existing tag

Pick the tag from the User parameters dropdown and click Add user param — it becomes a column you can see and query on. Load and refresh tags at this level pulls in the latest tag values for the current view level. Remove user param takes a column back off.

Create a new tag

Use the New user parameter panel (region 3):

Set the level first

The tag attaches to the current View level — a Dataset tag and a Cell tag are different things. Choose the level before creating.

Name, type, options, description

Give it a Name, a Type (e.g. string, number, or a fixed list of Options), and a Description so others know what it means.

Create

Click Create user param. This builds the UserParam… table — e.g. a Dataset tag called “DrugPresent” becomes sln_symphony.UserParamDatasetDrugPresent.

Fill in values

Add the new tag as a column, select rows in the Browser, and set the value for the current selection.

Query by a tag Once a tag exists you can search on it like any attribute — Add condition with user param adds it to the query so you can pull, say, every dataset where DrugPresent = true.

3 · Assign a cell type

At the Cell view level (with Curator mode off), select the cell(s), pick a Cell class and Cell type from the dropdowns, and click Assign cell type. This records the assignment against the cell’s permanent cell_unid as a typing event, so the history is kept. In the Details panel, unassigned means the cell has never been opened in Curator; other unclassified means it has.

4 · Run an analysis

Analyses compute results for your selected data. (Want to write one? See Writing an analysis.)

Select the data

Query and select the Datasets (most common) or Cells you want to analyze in the Browser.

Choose the analysis

Click Choose analysis and pick the .m file, or type its name (e.g. SMS_CA) if it’s on your path. Pick a Param set if the analysis takes parameters.

Set the switches

Selection vs all query results; Include raw data if the analysis needs traces; and Command line vs Database — use Database to store the results. Set Replace to All to overwrite existing results, or None to keep them.

Run analysis

Click Run analysis and watch the MATLAB Command Window for progress and any errors.

Before you run on the database If you changed anything in DJ_schwartzlab, commit or stash it first so the git_tag stored with each result is meaningful. Don’t merge branches unless you know what you’re doing.

5 · Explore results

To look at results that are already stored, tick Look for saved results and run your query at the level of the analysis (usually Dataset). Matching results appear in Results in database (region 5).

Pick a result

Select a dataset/cell in the Browser; its stored results list in Results in database. Compatible Plotters become available automatically.

Plot it

Choose a Plotter; use the stimulus-condition dropdowns to focus on particular conditions. The figure draws in the Plots panel.

Save the figure

Save single plot or Save all plots in folder, in PNG / PDF / FIG, with an optional multi-panel Layout. Plot Designer opens a finer editor.

6 · Export datasets & results

Three export paths, for three different needs:

  • Export Full DataGroup — the whole selected data group. Tick Include raw traces and/or Include spike times to bundle those in. The File / Command line rocker chooses between saving a .mat file and dropping data_group + data_group_struct into your MATLAB workspace.
  • Export Selection / Export Results — hand the current selection to the H5_Exporter for a structured HDF5 export.
  • To HDF5 — save the one stored result you’ve selected in Results in database to an .h5 file. To command line sends it to the workspace instead.

7 · The sub-tools: Curator & Spike Detector

At the Cell view level, the Details panel offers two buttons that open dedicated apps for the two jobs DataGrouper doesn’t do itself:

To Curator

Build a dataset

Pick the good epochs from a cell and save them as a named Dataset.

 To Spike Detector

Detect spikes

Turn the raw traces into spike times stored in SpikeTrain.

Restrict by selection narrows the view to what’s currently selected — select a cell, and moving to Epoch level shows just that cell’s epochs.

Save the query Use Save query to store the current query in sln_lab.Query under a name and project. Anyone can later re-run it from QueryBrowser.
