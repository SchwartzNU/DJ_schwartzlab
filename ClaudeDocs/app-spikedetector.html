Apps · detecting spikes

Spike Detector

Spike-based analyses need to know when each spike happened. The Spike Detector turns a cell’s raw traces into spike times and stores them in the SpikeTrain table. You open it from DataGrouper’s To Spike Detector button at the Cell level (it launches SpikeDetectorGUI_DB for the current cell).

Curate first If the cell has no cell-attached dataset epochs yet, you’ll get a “may need curation” prompt — build a dataset in the Curator first, then come back.

The workflow

It’s a threshold detector: the trace is band-pass filtered to emphasize spikes, then events crossing a threshold are marked. You set the threshold, check the result, clean up any noise by hand, and apply it across the cell’s epochs.

Pick a detection mode

The Spike detector mode menu offers Standard deviations above noise, Simple threshold, and advanced. Standard-deviations-above-noise is a good default because it adapts to each trace’s noise level.

Set the threshold

Type a value in Threshold (-9 is a common starting point, but it depends on your spike size), or click Click Threshold and pick the level directly on the trace.

Detect & check

Detect spikes marks events on the current epoch. Look them over; Clear spikes wipes them so you can retry. Step through epochs with Back 10 / Forward 10.

Fix noisy stretches

Use Select valid region and Select invalid region to keep or ignore parts of a trace — handy for perfusion artifacts, a lost seal, or a burst of noise that would otherwise be counted as spikes.

Apply across the cell

When one epoch looks right, Apply to all runs the same settings over every epoch, or Apply to this & future from the current one onward — useful when the recording changes partway through.

Save to the database

Tick Autosave to store as you go, or click Save & Sync to write the results to SpikeTrain (a spike_count and the spike_indices for each epoch).

Current-clamp shortcut Cell-attached recordings need this hand pass because their spikes aren’t detected automatically. Current-clamp spike detection, by contrast, can be run in bulk from the MATLAB Command Window with parpopulate(sln_results.SpikeDetectCC).
