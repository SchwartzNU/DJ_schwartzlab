Apps · detecting spikes

Spike Detector

Spike-based analyses need to know when each spike happened. Spike Detector turns raw traces into spike times and stores them in the SpikeTrain table. Reach it from DataGrouper’s To Spike Detector button, or run SpikeDetector directly.

Why it’s more than a threshold

A simple voltage threshold catches obvious spikes but also noise and artifacts. Spike Detector adds a second stage: it describes each candidate event by a few features, plots them in that feature space, and lets you cluster real spikes apart from noise — then correct anything the automation got wrong.

The workflow

Load a cell’s data

Set the rig/clamp with the Rig switch, choose the experiment, and Load. Cell-attached spikes aren’t detected automatically, which is why you’re here.

Filter and threshold

Adjust the Filter settings to clean the trace, then set the Threshold (around -9 is a common starting point, but it depends on your spike size). Turn the threshold on to grab candidate events.

Look at feature space

Under PCA settings, choose which principal components to plot on the X axis and Y axis. Real spikes tend to clump together, apart from noise. You can Add feature from an anonymous function or an m-file if the default features don’t separate cleanly.

Cluster

Cluster events runs density clustering (the epsilon and minimum points settings control how tight a group must be). Recluster spikes re-runs it after you change settings.

Hand-correct

Draw inclusion zone / Draw exclusion zone let you lasso points to force-keep or force-drop, with the Keep included / excluded points switches. Reset inclusion/exclusion list clears those overrides. Split after selected epoch handles a cell whose spike shape changes partway through.

Push to the database

When the spikes look right, Push to database writes them as SpikeTrain rows (spike indices and counts per epoch and channel). You can Save settings to file and Load settings from file to reuse a good configuration.

Current-clamp shortcut Current-clamp spike detection can be run in bulk from the MATLAB Command Window with parpopulate(sln_results.SpikeDetectCC), rather than one cell at a time here.
