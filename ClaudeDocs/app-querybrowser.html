Apps · saved queries

QueryBrowser

Every query saved from DataGrouper lands in one shared list. QueryBrowser is where you find one, re-run it, and pull its data into MATLAB — without rebuilding the query. Launch it with QueryBrowser in the Command Window.

What you’re looking at

The big table lists saved queries (the sln_lab.Query table). Each row is one query — its name, who made it, its project, and, once fetched, how many experiments / animals / eyes / cells / datasets it returns. The Filter by strip at the top narrows the list by User and Project.

Re-run a query

Find it

Use the User and Project filters to shrink the list, then click the query’s row to select it.

Fetch the datasets

Click Fetch. The query re-runs and its results are placed in your MATLAB workspace as datasets (and datasets_table), and the row’s count columns fill in.

Or fetch analysis results

Pick a result table from the Result table dropdown (the Dataset… tables from analyses) and click Fetch result. With Result to set to Command line, the results arrive as data / data_table in your workspace; set it to File to save instead.

Manage saved queries

With a row selected:

  • Set project — file the query under the project chosen in the Projects dropdown, so it’s easy to find later.
  • Rename — give the query a clearer name (it keeps its underlying SQL and project).
  • Delete — remove the saved query after a confirmation. This only deletes the saved query, not any data or results.
Where do saved queries come from? You create them in DataGrouper with Save query. QueryBrowser is the read/re-run side of the same sln_lab.Query table — a shared library of the lab’s useful searches.
