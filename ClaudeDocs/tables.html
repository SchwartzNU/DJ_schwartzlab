Every table, in plain language

A schema is a folder of related tables. Each card below is one table: what it’s called, what identifies a row, and what it stores. The colored stripe is its tier — the same color coding you’ll see in a DataJoint diagram inside MATLAB.

Stimulus protocol parameters

Protocol parameter tables ×330

Every stimulus protocol auto-generates two parameter tables per version: …bp (one row per epoch block — the settings held constant for the whole run) and …ep (one row per epoch — the value that changed on that trial), with a V1…V12 version suffix.

Don’t query the versioned tables directly. Reach them through the aka helpers, which union every version and join the block and epoch params for you — keyed so each epoch carries its block’s settings:

% All "spots multi size" params — every version, block + epoch joined:
params = aka.BlockParams('SpotsMultiSize') * aka.EpochParams('SpotsMultiSize');
params = aka.SMSparams;          % identical — the ready-made alias

% Restrict to one dataset's epochs, then fetch every parameter:
q   = params & (aka.Dataset & 'dataset_name = "..."');
tbl = fetch(q, '*');

aka.BlockParams('<Name>') and aka.EpochParams('<Name>') work for any protocol. Many also have a ready-made combined alias: aka.SMSparams, aka.MovingBarparams, aka.ContrastResponseParams, aka.DriftingGratingsparams, aka.FlashedBarParams, aka.LSparams, aka.MovingObjectparams, aka.MultiPulseParams, aka.PairedSpotsParams, aka.PulseParams, aka.RampParams. The <Name> is the protocol’s base name — the part between ExperimentProt and V<n> in the table names below.
