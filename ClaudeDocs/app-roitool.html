Apps · calcium imaging

ROI Tool

For functional (calcium) imaging, the signal you care about lives in regions of interest — usually cell bodies. ROI Tool is where you define those regions on an imaging run and pull out each one’s fluorescence-over-time trace. Launch it with ROI_Tool in the Command Window.

What it works on

ROI Tool operates on a functional-imaging run — a sln_funcimage.ImagingRun, which is a movie of frames over time. The Imaging run information panel shows that run’s frame rate, duration, pixel resolution, and physical size. Load one from the database with the Load from database dropdown, or open a .tif movie plus its metadata from File → Load image.

The workflow

Make a projection

A single frame is noisy, so first flatten the movie into one image. In the Projections panel choose Mean, Max, or Std projection — the std projection highlights pixels that change, which is what active cells do. Flip Limit to time sliders to project only the window you’ve selected. Then set the Threshold slider and Apply to turn the projection into a mask of candidate pixels.

Find ROIs automatically

In the Correlations panel, Compute all correlations (this needs a projection mask first) measures how similarly nearby pixels behave over time. Set the Correlation threshold, Search radius, path-distance, and Area thresholds, then Make ROIs — neighbouring pixels whose traces move together get grouped into one ROI.

Or draw them by hand

The Select menu switches drawing modes: Freedraw to lasso a region, or Point to drop a marker. Handy for cleaning up or adding cells the automatic pass missed.

Check the traces

Each ROI’s fluorescence-over-time trace is drawn on the right, alongside an ROI heat-map. Use the High pass (Hz) field to filter slow drift out of the traces, and the time sliders to scrub through the movie. Every action is logged in Processing steps; Clear last / Clear all undo them.

Grating responses (optional)

For drifting-grating runs, pick a Grating angle and Plot ROI amplitude map to see how strongly each ROI responded at that direction — a quick read on orientation / direction tuning.

Save the ROIs

Write ROIs to database records a sln_funcimage.ROIMethod (tagged ROI_Tool, with your processing steps as notes) and stores the ROIs against the imaging run, feeding sln_funcimage.ROIField and ROITraces. To keep a local copy instead, File → Save ROIs writes a .mat plus a mask .tif.

Where the results live ROIs and their traces land in the sln_funcimage schema, so downstream analyses and QueryBrowser can pull them like any other result.
